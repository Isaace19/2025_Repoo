Introduction

In C++, you could easily include vector and have a fully featured vector class that allows you to push items, insert items, and remove items. The implementation is all taken care of for you, including dynamic memory allocation as well as ensuring that all items are in contiguous memory.

Unfortunately, C doesnt have anything like that. Furthermore, the CPP version of vector took a template, which was the data type you wanted to store in the vector. C, again, has no concept of a template. However, just like CPP, we can manage memory agnostic of the data type.

Vectors

Vectors must maintain the properties: that (1) the memory is in order and (2) the memory is contiguous. This complicates insertions and deletions since the memory is only contiguous at first. When you insert an item, you must recreate all of the memory, copy the old values into the new memory, and free the old memory.

Assignment

There are three files: (1) main.c, (2) vector.c, and (3) vector.h. You will create an opaque vector structure and put it in vector.c. main.c contains a default testing program, but it will not influence grading. The only file that will be tested is vector.c.

Vector Structure

The vector will be opaque, and it has the following structure:

size_t size;
size_t capacity;
int64_t *values;
Do not change the type or order of these fields. Since your structure is opaque, the grade script needs to reproduce the structure.

vector_new

Vector *vector_new(void);

This function will create a new, empty vector. Recall that the Vector structure is opaque, meaning all the user can see is the pointer. Therefore, this function must create the Vector on the heap. The capacity of this new vector will be 0. The values pointer will still be valid, even though it points to memory the size of 0. Do not set values to NULL.

vector_new_with_capacity

Vector *vector_new_with_capacity(size_t capacity);

Creates a new, empty vector, but creates memory enough for capacity number of elements. Since every value is an int64_t, you must create enough memory for 8capacity8capacity. The size field will determine the number of actual elements, which in this case, will be 0.

Do not overthink this function. Recall that size is the total number of added elements, whereas the capacity is the number of slots allocated in memory to put things.

Even if capacity is 0, it needs to create a pointer for the values array. Recall that malloc still works even with a size of 0.

vector_free

void vector_free(Vector *vec);

Frees all elements in the vector. After this function is called, the vector must be considered invalid. Make sure you free both the values pointer as well as the Vector pointer itself.

vector_push

void vector_push(Vector *vec, int64_t value);

Pushes the provided value to the back of the vector. If there is enough capacity, this just writes to the next slot, given by the size field. If there is not, then the vector needs to be resized (see vector_resize).

vector_insert

void vector_insert(Vector *vec, size_t index, int64_t value);

Inserts the value at the given index or to the end of the vector if the location is >= the last element.

vector_remove

bool vector_remove(Vector *vec, size_t index);

This will remove the element at the given index. This function returns true if the element exists and was removed; it returns false otherwise.

vector_shrink

void vector_shrink(Vector *vec);

This will shrink the memory given to the vector such that size is equal to the capacity. This must reallocate memory if this is not already the case. The point of this function is to conserve memory. Recall that capacity is the amount of memory allocated to the vector, whereas size is the number of useful elements in the vector.

vector_get

bool vector_get(const Vector *vec, size_t index, int64_t *value);

Gets a value from the vector at the given index. A pointer to the value to set is in value. This value may be NULL, in which this function should just return whether or not there is a value at the given index. This function returns true if index contains a valid value, false otherwise. The pointer *value is irrelevant to the return value.

vector_get_unchecked

int64_t vector_get_unchecked(const Vector *vec, size_t index);

Gets a value from the vector at the given index. Does not check the bounds of the index before attempting to retrieve it. Returns the value at the given index.

vector_set

bool vector_set(Vector *vec, size_t index, int64_t value);

Sets a value at the given index. Returns true if the index is within the size, false otherwise. This does NOT grow or shrink the vector.

vector_sort

void vector_sort(Vector *vec);

vector_sort_by

void vector_sort_by(Vector *vec, SortFunc comp);

The SortFunc type is defined as int comp(const void *left, const void *right), which is the function type you will need to use qsort to quick sort the elements. Recall that qsort sorts contiguous memory based on the comparator. The comparator returns -1 (left is less than right), 1 (left is greater than right), or 0 (left is equal to right).

vector_bsearch

ssize_t vector_bsearch(const Vector *vec, int64_t value);

Uses a binary search to find the given value. This assumes that the vector is sorted in ascending order. Returns the index of the value, if found, or -1 if the value is not found.

Use bsearch in the standard library. You will have to do a little bit of math since bsearch gives you a pointer to the element, and not its index. However, if you remember the formula from COSC230 (base + index * size), you can work it backwards to determine index.

vector_find

ssize_t vector_find(const Vector *vec, int64_t value);

Finds value using a linear search. Returns the index of the first occurrence of the value, -1 if the value is not found.

Use lfind in the search.h library. Just like bsearch, you will be given a pointer (or NULL if it cant be found).

vector_resize

void vector_resize(Vector *vec, size_t new_size);

Manipulates the size of the vector (and capacity if size > capacity). If the new size is bigger than the capacity, the vector must be reallocated, copied, and the old vector freed, then the capacity and size will be equal to each other.

vector_reserve

void vector_reserve(Vector *vec, size_t new_capacity);

Changes the capacity of the vector. If the capacity is smaller than the size, then the capacity will be set to the size. If the capacity is bigger than the size or previous capacity, then the vector must be reallocated, but the size field will remain unchanged.

vector_clear

void vector_clear(Vector *vec);

Clears the vector by setting the size to 0. This does not change the capacity nor does it free any memory. If the user wants to shrink the memory to 0, they will have to call vector_shrink.

vector_capacity

size_t vector_capacity(const Vector *vec);

Returns the capacity of the given vector.

vector_size

size_t vector_size(const Vector *vec);

Returns the number of elements in the given vector. This will be used frequently to loop through the elements of your vector.

Contiguous Memory

As you know, a vector requires that its elements be searchable by O(1) based on its index. This is done through the base+indexsizebase+indexsize formula, where the base is the memory location of the first byte. Growing or shrinking a vector requires about 2x the amount of memory.

The difference between growing and shrinking is the number of values copied. To grow, we copy all of the old values, but to shrink, we only copy up to the new size.
